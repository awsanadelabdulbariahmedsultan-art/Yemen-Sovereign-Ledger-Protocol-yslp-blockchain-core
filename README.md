## 🌍 Project Overview (English)
The **CBOYD** engine is a blockchain-driven sovereign framework designed to unify local Yemeni e-wallets (e.g., M-Floos, Jaib, Jawali, OneCash) and bridge them with global liquidity platforms, digital exchanges (Binance, Bloomberg), and international financial institutions (World Bank, IMF).

At its core runs the **Yemen Sovereign Ledger Protocol (YSLP)** built from scratch using **Go (Golang)**. This protocol consolidates civilian banking, industrial automation, maritime logistics, and refinery IoT systems into a singular, highly secure immutable ledger governed by the Central Bank of Yemen.

### 🛠️ Key Architectural Features:
1. **Sovereign Consensus (Proof of Authority - PoA):** Zero random gas fees, high throughput (thousands of TPS), and validator node exclusivity restricted to the Central Bank.
2. **Asymmetric Cryptography (ECDSA P-256):** Complete wallet protection utilizing private/public key pairs. Transactions require explicit user signature verification (biometric/private key) to prevent tampering.
3. **Multi-Asset Swap & Clearing Engine:** Instantaneous fiat-to-crypto (YER/USD to USDT/BTC) automated clearing using real-time global exchange rates.
4. **Industrial IoT & NFC Integration:** Automated payment settlements triggered by hardware taps on factory production lines, refinery valves/tanks, and port maritime shipping berths.

## 🚀 Project Architecture Overview (English Extension)
The **CBYD** engine bridges international traditional banking rails (**Fiat Clearing**) seamlessly with decentralized Web3 networks. The hybrid core allows simultaneous settlement between global multi-chain modules and central trade finance channels.

### 🌐 1. Advanced Multi-Chain Network Module
Autonomous cross-verification infrastructure routing assets across premier cryptographic ledgers directly to verified user destinations:
* **Base Network (Coinbase Layer-2):** Primary execution framework for instant, low-latency micro-settlements.
* **BNB Smart Chain (BEP-20):** Integrated liquidity bridge optimized for Binance ecosystem settlements.
* **Ethereum Mainnet (ERC-20):** Decentralized enterprise clearing environment for institutional smart contracts.
* **Bitcoin Mainnet (Taproot):** Native UTXO cryptographic verification utilizing `bc1p...` secure ledger nodes.
* **Tron Network (TRC-20):** High-volume rails deployed for mass stablecoin liquidity transfers.

### 🚢 2. Institutional Maritime Logistics & Factory Settlements
Automated deployment of **Maritime Freight Clearing Reports** linked directly to automated IoT sensors, manufacturing plants, and central custom clearings. It natively hosts technical data inputs for top maritime container carriers:
* **MSC** (Mediterranean Shipping Company - Switzerland)
* **Maersk Line** (A.P. Moller-Maersk - Denmark)
* **CMA CGM** (France)
* **COSCO Shipping Lines** (China)
* **Integrated Shipping Inputs:** Bill of Lading (`B/L`), Container Serial ID validation, route tracking (`POL: CNSHA` Shanghai ➔ `POD: YEADE` Aden), Delivery Orders (`D/O`), and international trade terms (`FOB`, `CIF`, `EXW`).

### 💳 3. Omni-Channel Global Payment Railings
Dynamic data tokenization capturing legacy electronic clearing cards alongside consumer FinTech infrastructure pipelines:
* **Traditional Bank Accounts:** International multi-currency Wire/Swift channels mapping Account numbers and international `IBAN` architectures.
* **Credit Clearing:** Secured tokenization protocols for Visa and Mastercard clearing.
* **Modern FinTech Sub-Gateways:** Native API integrations supporting **Tamara** (BNPL credit splits), **PayPal Global** (escrow and automated dynamic invoicing ID checks), and **WeChat Pay** (direct Chinese supply chain factory wallet interface).

## ⚡ Deployment & Testing / طريقة التشغيل والاختبار
To execute the unified simulator nodes and benchmark cross-border trade transactions, execute the following commands on your server:

1. Ensure **Go (Golang)** version 1.18+ or later is installed on your mobile environment/shell.
2. Bypass environment modules restriction locally:
   ```bash
   export GO111MODULE=off
   ```
3. Boot up the unified multi-asset settlement hub server:
   ```bash
   go run mobile_app_simulator.go
   ```
4. Access the smartphone payment deployment instance node via your default browser at: `http://localhost:8080`.
